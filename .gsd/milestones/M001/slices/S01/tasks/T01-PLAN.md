# T01: Monorepo Root Config + Workspace Stubs

**Slice:** S01 — Monorepo + Worktree Scaffold
**Milestone:** M001

## Description

Creates the complete structural skeleton of the monorepo: root workspace config, TypeScript base config, and all 9 workspace `package.json` + `tsconfig.json` stubs. No implementation code — just the named, typed shells that every downstream slice fills in.

## Steps

1. Create root `package.json`: `private: true`, `name: monster`, `version: 0.0.1`, `packageManager: pnpm@10.30.3`, empty `scripts` block, no dependencies.
2. Create `pnpm-workspace.yaml`: `packages: ['apps/*', 'packages/*']`.
3. Create `.npmrc`: `shamefully-hoist=false` and `strict-peer-dependencies=false` (avoids noise on stubs with no deps yet).
4. Create `tsconfig.base.json`: `compilerOptions` with `strict: true`, `target: ES2022`, `lib: [ES2022]`, `moduleResolution: Bundler`, `esModuleInterop: true`, `skipLibCheck: true`, `resolveJsonModule: true`. No `include`/`exclude` — each package sets those.
5. Create each app workspace directory with `package.json` and `tsconfig.json`:
   - `apps/admin/`: name `@monster/admin`, private. tsconfig: extends `../../tsconfig.base.json`, `moduleResolution: Bundler`, `plugins: [{name: "next"}]` (Next.js plugin added now so S04 can install Next.js without changing the tsconfig structure).
   - `apps/generator/`: name `@monster/generator`, private. tsconfig: extends `../../tsconfig.base.json`, `moduleResolution: Bundler`.
6. Create each package workspace directory with `package.json` and `tsconfig.json`:
   - All 7 packages (`db`, `shared`, `agents`, `analytics`, `domains`, `seo-scorer`, `deployment`): name `@monster/<pkg>`, `version: 0.1.0`, `private: true`, `main: dist/index.js`, `types: dist/index.d.ts`, empty `exports` object. tsconfig: extends `../../tsconfig.base.json`, `module: NodeNext`, `moduleResolution: NodeNext`, `outDir: dist`, `rootDir: src`, `declaration: true`, `declarationMap: true`. Include `src/**/*`.
7. Run `pnpm install` to generate lockfile and verify all workspaces resolve.

## Must-Haves

- [ ] Root `package.json` has `"private": true` and `"packageManager": "pnpm@10.30.3"`
- [ ] `pnpm-workspace.yaml` present with `apps/*` and `packages/*` globs
- [ ] All 9 workspace directories have `package.json` with correct `@monster/<name>` and `"private": true`
- [ ] All 9 workspace directories have `tsconfig.json` extending the base
- [ ] `pnpm install` exits 0 from repo root
- [ ] `pnpm ls -r --depth 0` shows all 9 `@monster/*` packages
