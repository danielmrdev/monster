# T01: Install deps + scaffold Next.js build infrastructure

**Slice:** S04 ‚Äî Admin Panel Shell
**Milestone:** M001

## Description

The `apps/admin` directory has only a package.json stub and a placeholder `src/index.ts`. This task adds all Next.js 15 + Tailwind v4 + shadcn + Supabase deps, fixes the tsconfig, creates the minimal config files, and gets `pnpm --filter @monster/admin build` to exit 0 on a near-empty app.

The critical constraint: always run `pnpm install` from the monorepo root (`/home/daniel/monster/`), never from inside `apps/admin`. Running it inside the subdirectory breaks workspace symlinks for `@monster/db` and `@monster/shared`.

Do NOT use `shadcn init --template=next` ‚Äî that creates a fresh Next.js project and wipes package.json. Use `pnpm dlx shadcn@latest init -y --cwd apps/admin` on an already-configured Next.js project.

After shadcn init, immediately verify `@monster/db` and `@monster/shared` are still in package.json ‚Äî shadcn occasionally overwrites it.

## Steps

1. **Update `apps/admin/package.json`** ‚Äî add to `dependencies`: `next: ^15.0.0`, `react: ^19.0.0`, `react-dom: ^19.0.0`, `@supabase/ssr: ^0.9.0`, `@supabase/supabase-js: ^2.0.0`; add to `devDependencies`: `@types/node: ^22.0.0`, `@types/react: ^19.0.0`, `@types/react-dom: ^19.0.0`. Add scripts: `"dev": "next dev --port 3004"`, `"build": "next build"`, `"start": "next start --port 3004"`. Keep existing `@monster/db: workspace:*` and `@monster/shared: workspace:*` untouched.

2. **Run `pnpm install`** from `/home/daniel/monster/` (monorepo root).

3. **Fix `apps/admin/tsconfig.json`** ‚Äî override compilerOptions to add: `"lib": ["dom", "dom.iterable", "esnext"]`, `"jsx": "preserve"`, `"module": "esnext"`, `"noEmit": true`, `"isolatedModules": true`, `"allowJs": true`, `"skipLibCheck": true`. Add `"next-env.d.ts"` to the `include` array alongside `"src/**/*"`. Keep `"moduleResolution": "Bundler"` and the next plugin entry.

4. **Write `apps/admin/next.config.ts`**:
   ```ts
   import type { NextConfig } from 'next'
   const nextConfig: NextConfig = {}
   export default nextConfig
   ```

5. **Write `apps/admin/postcss.config.mjs`**:
   ```js
   export default { plugins: { '@tailwindcss/postcss': {} } }
   ```

6. **Delete `apps/admin/src/index.ts`** (the placeholder from S03). Write a minimal `apps/admin/src/app/globals.css`:
   ```css
   @import "tailwindcss";
   ```
   Write a minimal `apps/admin/src/app/layout.tsx` (no auth yet ‚Äî just bare HTML shell):
   ```tsx
   import './globals.css'
   export default function RootLayout({ children }: { children: React.ReactNode }) {
     return <html lang="en"><body>{children}</body></html>
   }
   ```
   Write a minimal `apps/admin/src/app/page.tsx`:
   ```tsx
   export default function Home() { return <div>BuilderMonster Admin</div> }
   ```

7. **Run `pnpm dlx shadcn@latest init -y --cwd apps/admin`** from the monorepo root. Respond to any interactive prompts with defaults. After it completes, verify `apps/admin/components.json` exists. Then verify `@monster/db` and `@monster/shared` are still in `apps/admin/package.json` ‚Äî if removed, re-add them with `workspace:*`.

8. **Run `pnpm install`** again (shadcn may have added deps). Then run `pnpm --filter @monster/db build && pnpm --filter @monster/shared build` to ensure dist artifacts are current. Then run `pnpm --filter @monster/admin build`.
