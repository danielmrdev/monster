# S03: CategoryForm Meta + Homepage SEO

**Goal:** Add a `meta_description` field to CategoryForm (saving to `tsa_categories.description` per D057), and add a Homepage SEO section to the site edit page with `focus_keyword` + `homepage_seo_text` editor + Generate with AI.
**Demo:** Edit a category — a "Meta Description" textarea appears and saves. Open site edit — a "Homepage SEO" card shows `focus_keyword` (already present) and a new `homepage_seo_text` textarea with Generate with AI button.

## Must-Haves

- `CategoryForm` renders a `meta_description` textarea; save writes to `tsa_categories.description` column (D057)
- Category edit `defaultValues` populates `meta_description` from the `description` column
- Site edit page has a "Homepage SEO" card with `focus_keyword` + `homepage_seo_text` textarea
- `updateSite` action saves `homepage_seo_text` to `sites.homepage_seo_text` column
- Generate with AI on the homepage SEO section streams into the `homepage_seo_text` textarea
- `generate-seo-text` route handles `homepage_seo_text` field case

## Observability / Diagnostics

- **CategoryForm save:** If `updateCategory` fails, Supabase error is returned as `errors._form[0]` and displayed in the form. Check the server console for the raw Supabase error object.
- **Inspect saved value:** `pnpm --filter @monster/db supabase db query "SELECT id, description FROM tsa_categories WHERE id='<catId>'"` — confirms the `meta_description` value persisted.
- **Homepage SEO generation failures:** The generate-seo-text SSE stream emits `{"type":"error","error":"<message>"}` events; the UI displays them below the textarea.
- **Build errors:** `pnpm --filter @monster/admin typecheck` — TypeScript errors point directly to mismatched prop types between CategoryForm interface and edit page defaultValues.
- **Failure state visibility:** If `updateSite` or `updateCategory` returns `errors._form`, the form renders a red banner. No silent swallowing.

## Verification

- Edit a category, add meta description, save → `SELECT description FROM tsa_categories WHERE id=?` returns the saved value
- Edit a site, add homepage_seo_text, save → `SELECT homepage_seo_text FROM sites WHERE id=?` returns the saved value
- `pnpm --filter @monster/admin build` exits 0
- **Failure path:** Submit CategoryForm with empty name → form returns validation error visible in UI (not a silent 500)
